# Blackstone Knowledge Registry (BKR)

## Volume V — Canonical Definitions, Taxonomies, and Governance Records

### Version 2.0

---

## Preamble

This volume establishes the shared vocabulary, canonical identifiers, controlled taxonomies, and governance records that all other volumes depend upon. It is the registry layer of the Blackstone Ecosystem: stable, referenceable, and versioned.

The Blackstone Knowledge Registry (BKR) does not contain every knowledge object produced by the ecosystem. It contains the definitions, schemas, and records that make consistent knowledge governance possible across Hermes, Blackstone, SintraPrime-Unified, Mission Control, and future agents.

## 2. Identifier Standards

### 2.1. Knowledge Object Identifier

Format: `BKO-{domain}-{NNNNNN}`

- `domain` — short subject-domain code (e.g., `LEGAL`, `FIN`, `TAX`, `MED`).
- `NNNNNN` — zero-padded six-digit sequence.

Example: `BKO-LEGAL-000001`

### 2.2. Evidence Identifier

Format: `EV-{NNNNNN}`

Example: `EV-000001`

### 2.3. Reasoning Record Identifier

Format: `RR-{NNNNNN}`

Example: `RR-000001`

### 2.4. Constitutional Decision Record Identifier

Format: `CDR-{YYYY}-{NNNN}`

- `YYYY` — ratification year.
- `NNNN` — zero-padded four-digit sequence.

Example: `CDR-2026-0001`

### 2.5. Source Registry Identifier

Format: `SRC-{domain}-{NNNN}`

Example: `SRC-IRC-0001`

### 2.6. Jurisdiction Identifier

Format: `JUR-{code}`

- `code` — jurisdiction code from the controlled list.

Example: `JUR-US-FEDERAL`

### 2.7. Subject Area Identifier

Format: `SUBJ-{code}`

Example: `SUBJ-TAX-PROCEDURE`

### 2.8. Agent Identifier

Format: `AGENT-{name}-{version}`

Example: `AGENT-HERMES-2-0`

### 2.9. Certification Identifier

Format: `CERT-{YYYYMMDD}-{NNNN}`

Example: `CERT-20260706-0001`

### 2.10. Regression Identifier

Format: `REGRESSION-{YYYYMMDD}-{NNNN}`

Example: `REGRESSION-20260706-0001`

| Code | Label | Definition |

| `GOVERNMENT_PRIMARY` | Government Primary | Official government records, statutes, regulations, court filings. |
| `JUDICIAL` | Judicial | Court opinions, orders, and rulings. |
| `LEGISLATIVE` | Legislative | Statutes, bills, legislative history. |
| `ADMINISTRATIVE` | Administrative | Agency guidance, rules, rulings, and interpretations. |
| `ACADEMIC` | Academic | Peer-reviewed scholarship, law reviews, university publications. |

## 4. Claim Taxonomy Registry

### 4.1. Controlled Claim Classifications

| Code | Label | Definition |
|------|-------|------------|
| `CONTROLLING` | Controlling | Supported by governing legal authority within the relevant jurisdiction. |
| `PERSUASIVE` | Persuasive | Supported by respected but non-controlling authority. |
| `HISTORICALLY_DOCUMENTED` | Historically Documented | Supported by historical evidence but not necessarily current law. |
| `SCHOLARLY` | Scholarly | Derived primarily from academic analysis. |
| `EDUCATIONAL` | Educational | Intended for learning and explanation. |
| `EMERGING` | Emerging | Supported by limited evidence requiring further validation. |
| `DISPUTED` | Disputed | Credible authorities materially disagree. |
| `UNVERIFIED` | Unverified | Insufficient evidence currently available. |

### 4.2. Multi-Status Combinations

A claim may carry multiple classifications. Common combinations include:

- `CONTROLLING` + `EDUCATIONAL` — a controlling authority used for teaching.
- `SCHOLARLY` + `HISTORICALLY_DOCUMENTED` — academic analysis of historical law.
- `PERSUASIVE` + `DISPUTED` — a respected but non-controlling view that other authorities reject.
- `EMERGING` + `UNVERIFIED` — early-stage claim with very limited evidence.

### 4.3. Claim Registry Entry Template

```markdown
# CLAIM-TAX-0001 — CNC Hardship Eligibility

**Claim:** A taxpayer may request currently not collectible status based on economic hardship.
**Classifications:** CONTROLLING
**Jurisdiction:** JUR-US-FEDERAL
**Subject:** SUBJ-TAX-PROCEDURE
**Confidence:** HIGH
**Maturity Stage:** OPERATIONAL
**Evidence:** EV-000001, EV-000002
**Counter-Evidence:** EV-000003
**Related Claims:** BKO-LEGAL-000010
**Status:** ACTIVE
```

## 7. Confidence and Maturity Registry

### 7.1. Controlled Confidence Levels

| Code | Label | Criteria |
|------|-------|----------|
| `HIGH` | High Confidence | Strong, corroborated evidence and controlling or persuasive authority. |
| `MODERATE` | Moderate Confidence | Substantial evidence, but with gaps or non-controlling authority. |
| `LIMITED` | Limited Confidence | Some supporting evidence, but significant uncertainty or limited corroboration. |
| `PRELIMINARY` | Preliminary Assessment | Early-stage evaluation; further evidence is expected to change the conclusion. |
| `INSUFFICIENT` | Insufficient Evidence | Not enough evidence to support a meaningful conclusion. |

### 7.2. Controlled Maturity Stages

| Code | Label | Description |
|------|-------|-------------|
| `IDEA` | Idea | An initial proposition or hypothesis, often unverified. |
| `HYPOTHESIS` | Hypothesis | A testable proposition with at least one identified evidence pathway. |
| `RESEARCH` | Research | Active evidence collection and evaluation underway. |
| `CORROBORATED` | Corroborated | Supporting evidence exists and has been reviewed. |
| `VERIFIED` | Verified | Evidence and verification meet the standards of the relevant governance process. |
| `OPERATIONAL` | Operational | Approved for use in supported workflows under appropriate human oversight. |
| `LITIGATION_READY` | Litigation-Ready | Meets heightened standards for legal proceedings. |
| `HISTORICAL_ARCHIVE` | Historical Archive | Superseded or no longer operational, but preserved. |

## 9. Constitutional Decision Record Registry

### 9.1. CDR Registry Rules

- Every ratified CDR is stored in `registry/cdr/`.
- CDRs are immutable once ratified.
- Superseded CDRs remain in the registry; their status is updated to `SUPERSEDED` and a link to the superseding CDR is added.

### 9.2. Example CDR Entry

```markdown
# CDR-2026-0001 — AI-Generated Summaries Are Not Primary Authority

**Question:** Should AI-generated summaries be treated as primary authority?

**Decision:** No.

**Reasoning:** AI summaries are derivative works. They may assist research, but they do not originate legal authority and may omit contrary authority. Treating them as primary authority would violate the Provenance and Authority Governance Doctrines.

**Evidence:** BKGC Article VIII, BKGC Article IX, BKGC Article XXI

**Approved By:** Governance Board

**Effective Date:** 2026-07-06

**Status:** RATIFIED

**Supersedes:** None
```

## 12. Registry Maintenance

12.1. New entries require a CDR if they introduce a new controlled code, schema field, or agent type.

12.2. Corrections to registry entries are versioned and logged; deletions are rare and require a CDR.

12.3. The Evidence Curator is responsible for taxonomy consistency.

12.4. The Governance Board ratifies changes to controlled vocabularies.
